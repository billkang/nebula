# 测试覆盖四维指引

## 概述

生成测试用例时，需从以下四个维度覆盖被测功能。至少覆盖三个维度（包含"正常流程"和"验收标准"为最低要求）。

---

## 维度一：正常流程

覆盖功能的预期使用路径，即 "Happy Path"。

**关键问题：**
- 用户按预期操作时会发生什么？
- 最常见的成功路径是什么？
- 输入有效数据时系统如何响应？

**示例：**
```
ID: TC-login-001 | 类型: 正常流程
前置条件: 用户已注册且账号正常
测试步骤: 1. 输入正确用户名和密码 → 2. 点击登录
预期结果: 成功登录，跳转到首页
```

---

## 维度二：边界条件

覆盖输入值、状态或数据在极限条件下的行为。

**关键问题：**
- 输入允许的最大值/最小值？
- 字符串为空或达到最大长度？
- 列表为空的边界？
- 数值为 0 或负数？
- 时间边界（过去/未来/并发）？

**示例：**
```
ID: TC-login-002 | 类型: 边界条件
前置条件: 用户已注册
测试步骤: 1. 输入空密码 → 2. 点击登录
预期结果: 提示"密码不能为空"，禁止提交
```

---

## 维度三：异常场景

覆盖可能出错的路径、系统故障或非预期操作。

**关键问题：**
- 网络中断或超时？
- 数据不存在或已被删除？
- 权限不足？
- 并发操作冲突？
- 输入格式错误或非法字符？
- 第三方服务不可用？

**示例：**
```
ID: TC-login-003 | 类型: 异常场景
前置条件: 用户已注册但账号被锁定
测试步骤: 1. 输入正确的用户名和密码 → 2. 点击登录
预期结果: 提示"账号已被锁定，请联系管理员"
```

---

## 维度四：验收标准验证

逐条验证 PRD 或 Jira 中定义的验收标准（Acceptance Criteria）是否满足。

**关键问题：**
- Jira Issue 中列出了哪些 AC？
- PRD 中有哪些功能验收点？
- 每条 AC 是否有对应的测试用例验证？

**示例：**
```
ID: TC-login-004 | 类型: 验收标准
前置条件: 用户已注册
测试步骤: 1. 输入正确凭据 → 2. 勾选"记住我" → 3. 点击登录 → 4. 关闭浏览器 → 5. 重新打开登录页
预期结果: 用户仍处于登录状态，无需重新输入密码（验证 AC-3: 记住登录状态）
关联验收标准: AC-3
```

---

## 覆盖检查清单

生成后，检查以下问题：

- [ ] **正常流程**：是否覆盖了至少一个完整成功路径？
- [ ] **边界条件**：是否检查了输入/状态的关键边界？
- [ ] **异常场景**：是否覆盖了至少一个出错路径？
- [ ] **验收标准**：是否每条 AC 都有对应测试用例？
- [ ] **至少三个维度**：覆盖度是否 ≥ 3/4？
